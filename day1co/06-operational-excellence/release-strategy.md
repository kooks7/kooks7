# Release Strategy (Safe Delivery)

## 원칙
- 배포는 기능 출시가 아니라 **리스크 관리**입니다.
- 특히 정산/결제/보안 변경은 “되돌릴 수 있는가”가 핵심입니다.

## 체크리스트(예시)
- [ ] 마이그레이션은 backward-compatible 인가?
- [ ] 롤백 시 DB/캐시 정합성이 유지되는가?
- [ ] 모니터링/알람이 준비되었는가?
- [ ] 배포 영향 서비스가 식별되었는가?

## 점진적 배포
- 기능 플래그/설정 기반으로 트래픽을 단계적으로 전환
- 메시징 전환처럼 인프라 변화가 큰 경우, “구 경로 유지 + 신 경로 병행” 전략을 선호

## 데이터 마이그레이션
- “한 번에”보다 “단계적/검증 가능”하게
- History 테이블/검증 스크립트/리포트를 통해 재현성을 확보
