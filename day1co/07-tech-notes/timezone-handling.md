# Tech Note — Timezone Handling in Settlement

정산에서 타임존 관련 이슈는 “언젠가” 반드시 발생합니다.  
특히 월 경계(start/end) 산정은 서비스/운영환경에 따라 쉽게 흔들릴 수 있습니다.

## 문제 유형

- 서버 런타임 TZ와 사업 기준 TZ가 다름
- “월”의 정의가 UTC 기준과 다름(예: KST 기준 1월의 UTC 범위)
- 멀티사이트 운영에서 사이트마다 TZ가 다름

## 접근

- 기간 산정 로직을 **공통 유틸**로 통일
- 도메인 코드에서 start/end 계산을 직접 하지 않도록 제한
- 테스트 코드에 대표 케이스를 넣어 회귀 방지
